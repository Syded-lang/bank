🔧 CREATE .ENV FILE MANUALLY ON HOSTINGER
═══════════════════════════════════════════════════════════════

The installer might have failed to create the .env file.
Let's create it manually via SSH.


## STEP 1: Create .env File via SSH
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Run these commands in Hostinger SSH terminal:

```bash
cd ~/public_html/core

# Create .env file with your database credentials
cat > .env << 'EOF'
APP_NAME=ViserBank
APP_ENV=production
APP_KEY=base64:REPLACE_THIS_WITH_GENERATED_KEY
APP_DEBUG=false
APP_URL=https://eastbridgeatlantic.com

LOG_CHANNEL=stack

DB_CONNECTION=mysql
DB_HOST=localhost
DB_PORT=3306
DB_DATABASE=u299375718_db
DB_USERNAME=u299375718_user
DB_PASSWORD=Fishpoder123

BROADCAST_DRIVER=log
CACHE_DRIVER=file
QUEUE_CONNECTION=sync
SESSION_DRIVER=file
SESSION_LIFETIME=120

REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS=null
MAIL_FROM_NAME="${APP_NAME}"

AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=

PUSHER_APP_ID=
PUSHER_APP_KEY=
PUSHER_APP_SECRET=
PUSHER_APP_CLUSTER=mt1

MIX_PUSHER_APP_KEY="${PUSHER_APP_KEY}"
MIX_PUSHER_APP_CLUSTER="${PUSHER_APP_CLUSTER}"

PURCHASECODE=BYPASSED-LICENSE-NO-VERIFICATION-REQUIRED
EOF

# Generate application key
php artisan key:generate

# Clear all caches
php artisan config:clear
php artisan cache:clear
php artisan view:clear
php artisan route:clear

# Rebuild cache
php artisan config:cache

# Set correct permissions
chmod 644 .env

echo "✅ .env file created successfully!"
```

## ALTERNATIVE: Use File Manager

5. Save the file

## TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### Still getting Error 500?

Check Laravel logs:
```bash
tail -100 ~/public_html/core/storage/logs/laravel.log
```

Common issues:
- "No application encryption key" → Run: php artisan key:generate
- "Access denied" → Wrong DB credentials in .env
- "Base table not found" → Database not imported

### Database not imported?

Check if tables exist:
```bash
mysql -u u299375718_user -p'Fishpoder123' u299375718_db -e "SHOW TABLES;"
```

If empty, import database:
```bash
cd ~/public_html/install
mysql -u u299375718_user -p'Fishpoder123' u299375718_db < database.sql
```
