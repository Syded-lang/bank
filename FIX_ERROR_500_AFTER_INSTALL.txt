🚨 ERROR 500 AFTER INSTALLATION - QUICK FIX
═══════════════════════════════════════════════════════════════

This error happens because Laravel has CACHED configuration files
that don't match your new database credentials from the installer.


## SOLUTION 1: Clear Cache via SSH (Fastest)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Login to Hostinger SSH and run these commands:

```bash
cd ~/public_html/core

# Clear all Laravel caches
php artisan config:clear
php artisan cache:clear
php artisan view:clear
php artisan route:clear

# Rebuild the cache with new settings
php artisan config:cache
```

After running these, refresh your browser at:
https://eastbridgeatlantic.com

## WHY THIS HAPPENS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Installer creates new .env file with database credentials
2. Laravel has OLD cached config files (from before installation)
3. Laravel tries to use cached config instead of new .env
4. Cache conflicts with new settings → Error 500

Clearing the cache forces Laravel to read the new .env file!

### Check Error Logs:

Via SSH:
```bash
tail -50 ~/public_html/core/storage/logs/laravel.log
```

Via File Manager:
Navigate to: public_html/core/storage/logs/
Open: laravel.log
Read the last 50 lines

Common errors:
- "Base table or view not found" → Database not imported
- "Access denied" → Wrong database credentials in .env
- "No application encryption key" → Run: php artisan key:generate


### Check Database Connection:

Via SSH:
```bash
cd ~/public_html/core
php artisan tinker
DB::connection()->getPdo();
exit
```

Should show: "PDO {#...}" (success)
If error: Check .env database credentials


### Check File Permissions:

```bash

```
